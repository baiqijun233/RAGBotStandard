Project001 RAG Bot Standard 项目解析

这份网页文档说明 Project001 的核心入口、模块分工、运行流程、本轮改造重点、当前边界和后续方向。它对应的主项目目录是 E:\Agent\AIProjects\Project001_RAGBotStandard。

一、项目定位

这个项目是一个本地 Streamlit PDF RAG 应用。用户上传 PDF 后，系统会读取文本、切分片段、写入 Chroma 向量库，再根据问题检索相关片段，并调用大模型生成中文回答。

二、核心入口是什么

核心入口是 app.py。它负责 Streamlit 页面、登录、上传 PDF、侧边栏配置、历史记录展示、调用向量检索、调用大模型、展示答案和来源。

cd E:\Agent\AIProjects\Project001_RAGBotStandard\02_Source\rag-bot-standard
.\.venv\Scripts\Activate.ps1
python -m streamlit run app.py

浏览器打开 http://localhost:8501 后，先进入登录页，再进入 RAG 问答页面。

三、运行流程

登录 -> 上传 PDF -> 提取文本 -> 切分 chunk -> 写入 Chroma -> 用户提问 -> similarity_search -> 组装 Prompt -> LLM 普通或 SSE 流式回答 -> 保存历史 -> 展示来源

四、模块作用

文件 作用 本轮改造点
app.py Streamlit 页面入口，负责把所有模块串起来。 新增登录页、聊天历史、SSE 开关、Chroma 目录展示、来源展示。
rag_bot/auth.py 读取登录配置并校验用户名密码。 新增本地轻量登录，支持环境变量覆盖默认账号。

rag_bot/rag_answering.py 把检索片段整理成中文 RAG Prompt，并处理无 Key 降级回答。 修复中文提示词乱码，保留非流式回答流程。
rag_bot/llm_client.py 调用 OpenAI 兼容 Chat Completions 接口。 新增 DeepSeek 默认配置、SSE 流式请求、流式内容解析。

五、改造重点

1. 从检索器升级为问答机器人

早期代码只把检索片段展示出来。现在会把多个片段组织成 Prompt，并交给大模型生成中文答案。

2. 从临时 Chroma 改为项目内持久化

Chroma 数据固定写入 04_Data/chroma，符合“所有项目相关内容都必须放在项目文件夹内”的规则。

3. 增加可测试的登录和历史记录

登录和历史记录没有直接写死在页面逻辑里，而是拆成小模块，方便测试和以后替换。

4. 增加 SSE 流式输出

LLM 客户端可以解析 OpenAI 兼容的 data: {...} 流式响应。页面提供开关，服务商不稳定时可以切回普通回答。

5. 修复乱码并补文档

本轮把 README、工作台、迁移说明、困难总结、网页解析文档重新整理为正常中文，并记录了本次遇到的问题。

八、当前边界

这个项目已经具备本地测试所需的核心能力，但登录仍是轻量门禁，历史记录仍是会话级保存，真实大模型回答还需要配置有效 API Key 后继续联调。
  • 历史记录不会跨服务重启长期保存。
  • 登录没有数据库账号系统，适合本地和轻量演示。
  • SSE 依赖服务商是否完整支持 OpenAI 兼容流式格式。
  • 没有 API Key 时不会产生真正的 AI 总结，只会显示检索依据。
